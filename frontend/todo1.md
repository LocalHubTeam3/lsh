  ## 우선 보완이 필요한 기능

  1. 장소 설명과 관련 커뮤니티 글

  문서는 장소 상세에서 위치, 설명, 이미지, 관련 커뮤니티 글을 확인하도록 제안합니다(frontend/reference.md:50).

  현재 frontend/src/views/LocationDetailView.vue:33는 다음 정보만 표시합니다.

  - 이미지
  - 장소명과 주소
  - 혼잡도
  - 카테고리
  - 데이터 ID
  - 지도 링크

  장소 설명과 해당 장소에 연결된 커뮤니티 글은 없습니다. 장소 카드의 “간단 설명”도 빠져 있습니다(frontend/reference.md:190).

  다만 백엔드 LocationOut에도 설명 필드가 없어서 프론트만 수정해서는 해결되지 않습니다. 백엔드에 overview 또는 description 데이터가 먼저 추가돼야 합니다.

  2. 커뮤니티 카테고리·장소 필터

  문서는 커뮤니티에 카테고리 또는 장소별 필터를 포함하도록 제안합니다(frontend/reference.md:211).

  현재 frontend/src/views/CommunityView.vue는:

  - 검색
  - 최신순/조회순 정렬
  - 페이지네이션

  만 제공합니다. 게시글에 카테고리는 표시되지만 필터로 선택할 수는 없습니다.

  백엔드 GET /api/posts도 현재 search, sort, page, size만 받기 때문에 카테고리 필터를 제대로 추가하려면 백엔드 쿼리도 확장해야 합니다.

  3. 게시글 작성 시 관련 장소 선택

  문서는 글 작성 화면에 관련 장소와 카테고리 선택을 제안합니다(frontend/reference.md:239).

  현재 frontend/src/views/PostWriteView.vue:11는 카테고리는 선택할 수 있지만 관련 장소를 연결할 수 없습니다.

  백엔드 PostCreate에도 location_id가 없으므로 다음 변경이 필요합니다.

  - 게시글 모델에 선택적 location_id 추가
  - 게시글 API 응답에 관련 장소 포함
  - 글쓰기 화면에 장소 검색·선택 UI 추가
  - 상세 화면에서 연결된 장소 표시
  - 커뮤니티 장소별 필터 추가

  이 기능을 구현하면 앞의 “관련 커뮤니티 글” 요구사항도 함께 해결할 수 있습니다.

  4. 작성자 또는 익명 표시

  문서는 작성자 표시명 또는 익명 처리를 요구합니다(frontend/reference.md:245).

  현재 작성자 입력은 없고, 게시글 목록과 상세에도 익명이라는 표시가 없습니다. 서비스가 익명 게시판이므로 백엔드 변경 없이 화면에 익명 라벨을 명시하는 정도로 보완할 수 있습니다.

  5. 맛집과 날씨

  문서는 지도·장소 영역에서 관광지, 맛집, 축제, 혼잡도, 날씨 등을 탐색하는 방향을 제안합니다(frontend/reference.md:35).

  현재 지원 분류는:

  - 관광지
  - 문화시설
  - 축제·행사
  - 여행코스
  - 레포츠
  - 숙박
  - 쇼핑

  맛집 데이터와 날씨 API는 없습니다. 특히 백엔드 원천 데이터에 음식점 파일이 제공되지 않았다는 기록이 있으므로 맛집 기능은 데이터 확보 없이는 구현하기 어렵습니다. 날씨 역시 별도 백엔드 API가 필요합니다.

  ## 부분 구현된 항목

  - 대표 지역 강조: 홈에서 서울을 강조하지만 사용자가 권역을 선택할 수는 없습니다.
  - 장소 카드 지도 버튼: 목록 상단에는 지도 진입 버튼이 있지만 개별 카드에는 상세 버튼만 있습니다.
  - 코스 기능: 장바구니에서 코스 생성은 가능하지만, 저장된 코스의 목록·상세·수정·삭제 화면은 없습니다. 코스는 reference 범위 밖에서 추가된 기능이지만 현재 기능 흐름은 완결되지 않은 상태입니다.
  - 지도 컴포넌트 분리: 문서는 LocationMap.vue 분리를 권장하지만 현재 지도, 상세 패널, 장바구니, 코스 폼이 모두 frontend/src/views/MapView.vue에 들어 있습니다. 기능 누락은 아니지만 유지보수 관점에서는 분리할 필요가 있습니다.

  ## 선택 기능이라 누락으로 보기 어려운 항목

  댓글은 문서에서도 “범위에 포함되면” 구현하도록 되어 있습니다(frontend/reference.md:232). 현재 백엔드에도 댓글 API가 없으므로 필수 누락으로 보기는 어렵습니다.

  로그인, 마이페이지, 업로드, 대시보드 역시 벤치마크 예시이며 LocalHub 필수 기능으로 지정된 것은 아닙니다.

  결론적으로 우선순위는 다음이 적절합니다.

  1. 게시글과 장소 연결
  2. 커뮤니티 카테고리·장소 필터
  3. 장소 설명 데이터 추가
  4. 익명 작성자 표시
  5. 저장된 코스 목록·상세 화면
  6. 맛집·날씨는 데이터/API 확보 후 구현
