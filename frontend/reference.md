# LocalHub Vue Frontend Reference

이 문서는 `C:\Users\SSAFY\Desktop\team_proj\2026-capstone-52\frontend`의 frontend를 벤치마크하여, 현재 `team_proj/frontend`를 Vue.js 3 SPA로 만들 때 참고할 구성, UX, 스타일 방향을 정리한 문서이다.

주의: 벤치마크 repo는 Vue가 아니라 `Next.js + React` 기반이다. 따라서 코드를 그대로 복사하지 않고, 화면 구조와 사용자 경험, 스타일 패턴만 참고해서 Vue 컴포넌트로 다시 작성한다.

## 목표 기술 방향

- Framework: Vue.js 3
- Build Tool: Vite 권장
- App Type: SPA
- Routing: Vue Router
- State: 우선 Composition API의 `ref`, `computed`, `reactive` 사용. 전역 상태가 커지면 Pinia 검토
- API: FastAPI REST API와 연동
- Deploy: Netlify

# 서울 여행 추천 서비스 프론트엔드 디자인 레퍼런스

## 1. 디자인 목표

서울의 관광지를 탐색하고 여행 코스를 제작·공유하는 서비스이다.

전체 디자인은 한국적인 분위기를 전달하되, 전통 요소를 과도하게 사용하지 않는다.
한옥, 한지, 단청, 낙관 등의 전통적인 시각 요소를 현대적인 지도 서비스 UI와 결합한다.

### 핵심 디자인 키워드

* 현대적인 서울
* 한옥의 문과 창호
* 한지 질감
* 절제된 단청 색상
* 여행 기록
* 친근한 호랑이 안내 캐릭터
* 깔끔하고 직관적인 지도 인터페이스

### 디자인 방향

```text
전통적인 한국 디자인 30%
현대적인 웹 서비스 UI 70%
```

서비스 전체가 민속촌이나 전통문화 체험 사이트처럼 보이지 않도록 한다.

한국적인 요소는 다음 위치에 제한적으로 사용한다.

* 랜딩 페이지의 대표 비주얼
* 페이지 제목 장식
* 버튼 또는 태그의 포인트 색상
* 로딩 화면
* 빈 화면 안내
* AI 분석 안내 캐릭터
* 코스 공개 완료 화면

## 2. 핵심 디자인 콘셉트

### 콘셉트 이름

**서울의 문을 열고 여행을 시작한다**

사용자가 랜딩 페이지에서 한옥 대문을 열고 서울 여행을 시작하는 듯한 경험을 제공한다.

### 주요 모티프

#### 한옥 대문

서비스를 대표하는 핵심 모티프이다.

사용 위치:

* 랜딩 페이지의 메인 이미지
* `코스 짜러 가기` 버튼 주변
* 페이지 전환 애니메이션
* 로고 심벌
* 모달 또는 패널 상단의 작은 장식

한옥 대문을 실제 UI 프레임 전체에 반복해서 사용하지 않는다.

#### 호랑이

호랑이는 서비스의 안내 캐릭터로 사용한다.

역할:

* 처음 방문한 사용자의 사용 방법 안내
* 여행바구니가 비어 있을 때 안내
* AI 코스 분석 중 로딩 화면
* 동선 경고 메시지
* 코스 저장 완료 메시지

호랑이가 UI보다 지나치게 눈에 띄지 않도록 작은 일러스트 또는 아이콘 형태로 사용한다.

예시 문구:

```text
가고 싶은 장소를 지도에서 골라보세요.
```

```text
선택한 장소를 바탕으로 동선을 살펴보고 있어요.
```

```text
이 코스는 하루에 방문하기 조금 빠듯할 수 있어요.
```

#### 갓과 한복

갓과 한복은 메인 디자인 요소로 사용하지 않는다.

다음과 같은 보조 요소로만 사용할 수 있다.

* 호랑이 캐릭터의 갓
* 사용자 기본 프로필 이미지
* 이벤트 배너의 작은 일러스트
* 404 페이지 또는 빈 화면 이미지

#### 낙관

붉은색 도장 형태의 낙관은 주요 CTA와 완료 상태에 사용한다.

사용 예시:

* 최종 확정 버튼
* 공개 완료 표시
* 추천 코스 배지
* 코스 카드의 인기 표시

버튼 전체를 도장 모양으로 만들기보다는 작은 아이콘이나 배지로 사용하는 것을 권장한다.

## 3. 전체 디자인 원칙

### 3.1 전통 요소는 배경이 아니라 포인트로 사용한다

한지 텍스처나 전통 패턴을 화면 전체에 강하게 적용하지 않는다.

좋은 예:

* 연한 한지 질감 배경
* 헤더 아래 얇은 단청 패턴
* 카드 모서리에 작은 전통 문양
* 제목 옆 작은 낙관 표시

피해야 할 예:

* 모든 버튼에 전통 문양 적용
* 모든 카드에 한옥 테두리 적용
* 화면 전체를 진한 단청색으로 구성
* 붓글씨 폰트를 본문에 사용

### 3.2 지도와 콘텐츠가 가장 먼저 보여야 한다

지도 페이지에서는 전통 장식보다 다음 요소가 우선되어야 한다.

1. 지도
2. 장소 마커
3. 상세 패널
4. 여행바구니
5. 카테고리 필터

전통적인 장식이 지도 사용성을 방해해서는 안 된다.

### 3.3 카드와 패널은 현대적인 형태를 유지한다

* 둥근 모서리
* 얕은 그림자
* 충분한 내부 여백
* 명확한 버튼 구분
* 읽기 쉬운 본문 글꼴

혼잡도는 색상만으로 구분하지 않고 텍스트를 함께 표시한다.

```text
● 혼잡
● 매우 혼잡
```

## 8. 랜딩 페이지

### 목표

사용자가 서비스를 처음 보자마자 다음 내용을 이해할 수 있어야 한다.

* 서울 여행 코스를 만들 수 있다.
* 장소를 지도에서 직접 고를 수 있다.
* AI가 동선을 분석한다.
* 만든 코스를 공유할 수 있다.

### 메인 화면 구성

```text
┌────────────────────────────────────────────┐
│                                            │
│          한옥 대문 또는 창호 비주얼        │
│                                            │
│     당신의 서울 여행 코스를 만들고,        │
│                 공유하세요                 │
│                                            │
│            [코스 짜러 가기]                │
│                                            │
└────────────────────────────────────────────┘
```

### 메인 문구

```text
서울의 문을 열고,
나만의 여행을 시작하세요.
```

보조 문구:

```text
가고 싶은 장소를 지도에서 고르면
AI가 여행 동선과 권장 일정을 분석해 드립니다.
```

버튼:

```text
[서울 코스 만들기]
```

### 비주얼 연출

한옥 대문 너머로 현대적인 서울 풍경이 보이는 구성을 권장한다.

예시 요소:

* 한옥 처마
* 남산서울타워 실루엣
* 서울 도심
* 한강
* 작은 호랑이 캐릭터

전통 건축만 보여주면 서울 전체 여행 서비스가 아니라 전통 관광 서비스처럼 보일 수 있으므로 현대 서울 요소를 함께 배치한다.

### 랜딩 페이지 추가 섹션

#### 서비스 사용 순서

```text
1. 장소를 고르세요
2. 순서를 정하세요
3. AI 분석을 확인하세요
4. 코스를 공유하세요
```

각 단계는 작은 일러스트와 함께 표시한다.

#### 인기 코스 미리보기

커뮤니티에 공개된 코스 중 일부를 카드로 노출할 수 있다.

```text
서울의 궁궐과 시장 하루 코스
성수와 서울숲 감성 코스
한강 야경 데이트 코스
```

## 9. 지도 페이지

지도 페이지는 서비스의 핵심 화면이다.

### 데스크톱 레이아웃

```text
┌──────────────────────────────────────────────────────┐
│ 헤더                                                 │
├───────────────┬──────────────────────────────────────┤
│ 장소 상세 패널 │                                      │
│               │                지도                  │
│               │                                      │
├───────────────┴──────────────────────────────────────┤
│ 여행바구니                                           │
└──────────────────────────────────────────────────────┘
```

### 지도 우선순위

지도는 화면에서 가장 넓은 영역을 차지해야 한다.

권장 비율:

```text
좌측 상세 패널: 320~380px
지도: 나머지 전체 영역
하단 여행바구니: 180~240px
```

### 카테고리 필터

지도 상단에 가로 형태로 배치한다.

```text
[관광지] [레포츠] [문화시설] [쇼핑] [숙박] [축제·공연·행사]
```

선택되지 않은 상태:

* 흰색 또는 한지색 배경
* 회색 테두리
* 먹색 텍스트

선택된 상태:

* 짙은 녹색 배경
* 흰색 텍스트
* 체크 아이콘

```text
[✓ 관광지]
```

여러 카테고리가 선택될 수 있음을 사용자가 알 수 있도록 체크 표시를 사용한다.

### 초기 빈 지도 안내

마커가 없는 초기 상태에서 지도 중앙에 큰 안내 모달을 표시하지 않는다.

지도 상단 또는 하단에 작은 안내 배너를 표시한다.

```text
카테고리를 선택하면 지도에 장소가 나타납니다.
```

호랑이 캐릭터를 작은 크기로 함께 사용할 수 있다.

## 10. 지도 마커

### 기본 마커

지도 핀에 한옥 기와 또는 창호 패턴을 약하게 적용한다.

카테고리마다 모양을 완전히 다르게 만들지 않는다.
마커 모양은 동일하게 유지하고 내부 아이콘 또는 색상만 변경한다.

### 카테고리 아이콘 예시

| 카테고리     | 아이콘      |
| -------- | -------- |
| 관광지      | 산 또는 기와  |
| 레포츠      | 달리는 사람   |
| 문화시설     | 전시관 또는 책 |
| 쇼핑       | 쇼핑백      |
| 숙박       | 침대       |
| 축제·공연·행사 | 깃발 또는 무대 |

### 클러스터 마커

클러스터는 원형으로 표시한다.

```text
  23
```

디자인:

* 짙은 녹색 배경
* 흰색 숫자
* 외곽에 한지색 테두리
* 확대 시 부드럽게 개별 마커로 전환

클러스터에 전통 문양을 복잡하게 넣지 않는다.

## 11. 장소 상세 패널

마커를 클릭하면 좌측에서 패널이 나타난다.

### 패널 애니메이션

```text
왼쪽에서 오른쪽으로 200~300ms 슬라이드
```

지나치게 긴 애니메이션은 사용하지 않는다.

### 정보 배치

```text
┌──────────────────────────┐
│ [대표 이미지]             │
│                          │
│ 장소 이름                 │
│ [관광지]                  │
│                          │
│ 현재 혼잡도               │
│ ● 보통                    │
│ 기준 지역: 광화문·덕수궁  │
│                          │
│ [여행바구니에 추가]       │
└──────────────────────────┘
```

### 이미지가 없는 경우

기본 이미지는 회색 박스보다 서비스 전용 일러스트를 사용하는 것이 좋다.

예시:

* 한옥 처마와 산
* 서울 스카이라인
* 지도 핀과 호랑이

표시 문구:

```text
등록된 대표 이미지가 없습니다.
```

### 드래그 가능한 카드 표현

카드 상단에 드래그 핸들을 표시한다.

```text
⠿ 여행바구니로 끌어다 놓으세요
```

버튼도 함께 제공한다.

```text
[여행바구니에 추가]
```

## 12. 여행바구니

여행바구니는 지도 하단에 고정된 패널로 구성한다.

### 접힘 상태

```text
여행바구니 3개                    [펼치기]
```

### 펼침 상태

장소 카드를 가로로 배치한다.

```text
[1 경복궁] → [2 북촌한옥마을] → [3 광장시장]
```

각 카드 정보:

* 순서 번호
* 대표 이미지
* 장소 이름
* 카테고리
* 삭제 버튼
* 드래그 핸들

### 방문 순서 표현

순서 번호를 낙관 형태의 작은 원으로 표현할 수 있다.

```text
① ② ③
```

또는 붉은색 원형 번호를 사용한다.

### 드래그 상태

카드를 드래그하는 동안:

* 카드 그림자 강화
* 원래 위치에 점선 자리 표시
* 이동 가능한 영역 강조
* 마우스 커서를 `grab`, 드래그 중에는 `grabbing`으로 변경

### 빈 상태

```text
아직 여행바구니가 비어 있습니다.

지도에서 장소를 선택한 뒤
장소 카드를 이곳으로 끌어오세요.
```

작은 호랑이 일러스트를 표시할 수 있다.

### 하단 버튼

```text
[전체 비우기]                         [AI 코스 분석]
```

`AI 코스 분석`은 장소가 2개 이상일 때 활성화하는 것을 권장한다.

장소가 부족할 때:

```text
AI 분석을 위해 장소를 2개 이상 추가해 주세요.
```

## 13. AI 코스 분석 UI

### 분석 중 화면

전체 페이지를 막는 로딩 화면보다 여행바구니 위에 분석 패널을 표시한다.

```text
호랑이가 여행 동선을 살펴보고 있어요.
```

로딩 애니메이션 후보:

* 지도 위를 움직이는 호랑이 발자국
* 붓으로 선이 그려지는 애니메이션
* 낙관이 찍히는 로딩 애니메이션

로딩 애니메이션은 1개만 선택한다.

### 분석 결과 패널

```text
┌──────────────────────────────────┐
│ AI 코스 분석                     │
│                                  │
│ 추천 방문 순서                   │
│ 1. 경복궁                        │
│ 2. 북촌한옥마을                  │
│ 3. 광장시장                      │
│                                  │
│ 권장 일정: 1일                   │
│                                  │
│ 서쪽에서 동쪽으로 이동하는       │
│ 순서로 방문하는 것이 좋습니다.   │
│                                  │
│ 주의사항                         │
│ 현재 선택한 장소는 오후 시간대에 │
│ 혼잡할 수 있습니다.              │
│                                  │
│ [추천 순서 적용] [닫기]          │
└──────────────────────────────────┘
```

### AI 정보 표시 원칙

AI 분석 결과와 확정된 사용자 선택을 명확하게 구분한다.

AI 결과에는 다음 문구를 표시한다.

```text
AI가 제안한 결과이며, 방문 순서는 자유롭게 변경할 수 있습니다.
```

AI가 추천 순서를 반환했다고 해서 자동으로 여행바구니 순서를 변경하지 않는다.

사용자가 다음 버튼을 눌렀을 때만 적용한다.

```text
[추천 순서 적용]
```

## 15. 코스 상세·공유 페이지

### 상단 영역

```text
서울 궁궐과 시장 하루 여행

서울나들이 님의 코스
추천 일정 1일 · 장소 3개

[좋아요] [공유 링크 복사] [SNS 공유]
```

### 대표 이미지

코스에 포함된 첫 번째 장소 이미지를 기본 대표 이미지로 사용한다.

추후 개선 시 여러 장소 이미지를 조합한 콜라주를 사용할 수 있다.

### 지도 영역

지도 위에 순서 번호 마커를 표시한다.

```text
1 → 2 → 3
```

장소 사이에는 단순한 직선을 표시한다.

지도 하단에 다음 안내 문구를 반드시 표시한다.

```text
장소 방문 순서를 나타낸 선이며 실제 이동 경로와 다를 수 있습니다.
```

### 장소 방문 순서

세로 타임라인 형태를 권장한다.

```text
① 경복궁
│
② 북촌한옥마을
│
③ 광장시장
```

타임라인 선은 붓 선처럼 약간 자연스러운 형태를 사용할 수 있지만 지나치게 거칠게 표현하지 않는다.

### AI 분석 영역

```text
AI 코스 분석

권장 일정
1일

코스 설명
서쪽에서 동쪽으로 이동하는 순서입니다.

주의사항
오후에는 일부 지역이 혼잡할 수 있습니다.
```

AI 분석 영역은 청색 계열의 연한 배경으로 일반 설명과 구분한다.

## 19. 좋아요 UI

좋아요는 하트 아이콘과 숫자를 표시한다.

```text

```

좋아요를 누른 상태:

```text
♥ 25
```

이미 좋아요를 누른 브라우저에서 다시 누르면 다음 메시지를 표시한다.

```text
이미 좋아요를 누른 코스입니다.
```

좋아요를 취소할 수 있도록 구현한다면 버튼을 다시 눌렀을 때 취소되도록 한다.

## 20. 반응형 디자인

* 중앙 지도
* 하단 여행바구니
* 카테고리 가로 배치

* 상세 패널 너비 축소
* 카테고리 가로 스크롤
* 여행바구니 높이 축소

```text
[여행바구니 3]
```

버튼을 누르면 전체 화면 또는 큰 하단 시트로 여행바구니를 표시한다.

### 모바일 카테고리

```text
[관광지] [문화시설] [쇼핑] →
```

한 줄 가로 스크롤 형태로 구성한다.

카테고리 버튼이 여러 줄로 내려가 지도 영역을 가리지 않도록 한다.

## 21. 애니메이션 원칙

애니메이션은 기능 이해를 돕는 경우에만 사용한다.

### 허용되는 애니메이션

* 상세 패널 슬라이드
* 카드 드래그 이동
* 마커 등장
* 클러스터 확대
* AI 분석 로딩
* 저장 완료 낙관 애니메이션
* 한옥 대문이 열리는 랜딩 애니메이션

### 권장 속도

```text
버튼 hover: 120~180ms
패널 전환: 200~300ms
모달 등장: 180~250ms
페이지 전환: 최대 400ms
```

한옥 대문 애니메이션은 최초 랜딩 페이지에서만 사용한다.

페이지 이동마다 대문이 열리고 닫히도록 만들면 사용성이 떨어질 수 있다.

## 22. 접근성

### 색상

* 텍스트와 배경의 명도 대비를 충분히 확보한다.
* 혼잡도를 색상만으로 표현하지 않는다.
* 선택된 카테고리에는 체크 아이콘을 추가한다.

### 키보드 조작

* 모든 버튼은 Tab 키로 이동 가능해야 한다.
* 드래그가 불가능한 사용자를 위해 순서 이동 버튼을 제공한다.

```text
[위로 이동] [아래로 이동]
```

* 모달이 열리면 포커스를 모달 내부에 유지한다.

### 이미지

모든 장소 이미지에 대체 텍스트를 설정한다.

```html
<img alt="경복궁 대표 이미지" />
```

기본 이미지:

```html
<img alt="장소 대표 이미지가 등록되지 않음" />
```

### 지도

지도만으로 정보를 전달하지 않는다.

선택한 장소는 여행바구니와 텍스트 목록에서도 확인할 수 있어야 한다.

## 25. 아이콘 사용 원칙

아이콘 스타일은 하나의 라이브러리로 통일한다.

권장 라이브러리:

* Lucide
* Heroicons
* Material Symbols

전통적인 아이콘이 필요하다면 별도 SVG로 제작한다.

별도 제작 후보:

* 한옥 대문
* 기와
* 낙관
* 호랑이 얼굴
* 창호 문양

이모지를 실제 서비스 아이콘으로 사용하지 않는다.

개발 초기 목업에서는 이모지를 임시로 사용할 수 있지만 최종 UI에서는 SVG 아이콘으로 교체한다.

## 26. 이미지 스타일

### 장소 이미지

* 카드 이미지 비율: `4:3`
* 코스 대표 이미지 비율: `16:9`
* `object-fit: cover`
* 이미지 위 텍스트 배치 최소화

### 일러스트 스타일

호랑이와 한옥 일러스트는 다음 기준을 따른다.

* 평면적인 벡터 스타일
* 색상 4~6개 이하
* 과도한 디테일 제거
* 둥글고 친근한 인상
* 캐릭터가 유아용 서비스처럼 보이지 않도록 조절

### 피해야 할 이미지

* 저해상도 전통 문양
* 서로 다른 그림체의 일러스트 혼용
* 실제 사진과 만화 캐릭터의 과도한 중첩
* 의미 없는 서울 랜드마크 나열
* 저작권이 불분명한 이미지

## 27. 빈 화면과 오류 화면

### 장소가 선택되지 않은 경우

```text
카테고리를 선택해 서울의 장소를 찾아보세요.
```

### 검색 결과가 없는 경우

```text
검색 결과가 없습니다.
다른 장소 이름이나 카테고리로 검색해 보세요.
```

### 여행바구니가 비어 있는 경우

```text
아직 선택한 장소가 없습니다.
지도에서 가고 싶은 장소를 골라보세요.
```

### AI 분석 실패

```text
코스 분석에 실패했습니다.
여행바구니의 장소는 그대로 유지됩니다.

[다시 분석하기]
```

### 코스 저장 실패

```text
코스를 저장하지 못했습니다.
입력한 내용은 유지되므로 잠시 후 다시 시도해 주세요.

[다시 저장하기]
```

### 404 페이지

갓을 쓴 호랑이가 지도를 거꾸로 들고 있는 일러스트를 사용할 수 있다.

```text
길을 잘못 든 것 같아요.

[홈으로 이동]
```

## 28. 알림 메시지

알림은 화면 오른쪽 위 또는 아래에 토스트 형태로 표시한다.

### 성공

```text
여행바구니에 추가했습니다.
```

```text
공유 링크를 복사했습니다.
```

```text
코스가 공개되었습니다.
```

### 경고

```text
이미 여행바구니에 있는 장소입니다.
```

```text
장소를 2개 이상 추가해 주세요.
```

### 오류

```text
장소 정보를 불러오지 못했습니다.
```

```text
댓글을 등록하지 못했습니다.
```

토스트에는 가능하면 작은 아이콘과 텍스트를 함께 표시한다.

## 31. 최종 디자인 권장안

서비스의 메인 전통 요소는 **한옥 대문**으로 결정한다.

호랑이는 다음과 같은 안내 상황에만 사용한다.

* 빈 화면
* 사용 방법 안내
* AI 분석
* 오류 또는 완료 메시지

갓과 한복은 호랑이 캐릭터 또는 기본 프로필 이미지에만 제한적으로 사용한다.

단청은 실제 전통 건축처럼 강한 원색으로 사용하지 않고, 다음 색상을 차분하게 변형해 사용한다.

* 녹색
* 붉은색
* 청색
* 황토색

### 전체 인상

```text
한옥의 따뜻함
+
서울 지도의 실용성
+
현대적인 여행 플랫폼의 깔끔함
```

### 가장 중요한 기준

1. 지도를 사용하는 데 불편함이 없어야 한다.
2. 여행바구니의 상태와 방문 순서가 명확해야 한다.
3. AI 추천과 사용자의 최종 선택을 구분해야 한다.
4. 한국적인 요소가 기능을 방해하지 않아야 한다.
5. 모든 페이지가 동일한 색상과 컴포넌트 규칙을 사용해야 한다.
6. 전통 요소를 많이 사용하는 것보다 일관되게 사용하는 것이 중요하다.
